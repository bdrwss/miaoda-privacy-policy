秒达 Multi-Account Session Isolation - 隐私政策
最后更新日期：2024年11月18日
1. 概述
秒达 Multi-Account Session Isolation（"本扩展"）是一个浏览器扩展，用于提供多账号会话隔离功能。我们非常重视您的隐私，本隐私政策说明了我们如何收集、使用和保护您的信息。

2. 数据收集
2.1 我们收集的数据
本扩展在您的设备本地存储以下数据：

用户配置数据：站点模板信息（名称、域名、登录URL等）、账号实例信息（账号名称、备注等）、用户设置和偏好
会话数据：Cookie数据（用于多账号隔离）、本地存储数据（localStorage、sessionStorage）、IndexedDB数据
技术数据：扩展使用情况（用于功能优化）
2.2 我们不收集的数据
❌ 个人身份信息（姓名、邮箱、电话等）
❌ 浏览历史记录
❌ 网站访问内容
❌ 密码或其他敏感凭证
❌ 位置信息
❌ 设备信息
3. 数据存储
3.1 本地存储
所有数据仅存储在您的本地设备上：

数据存储在浏览器的扩展存储空间（chrome.storage.local）
数据使用AES-GCM加密算法加密存储
加密密钥由用户主密钥通过PBKDF2派生
3.2 数据同步
本扩展不将数据同步到任何服务器
所有数据完全存储在您的本地设备
您可以通过扩展的备份功能导出数据
4. 数据使用
4.1 使用目的
我们使用收集的数据仅用于：

提供多账号会话隔离功能
管理不同账号的Cookie和存储
实现账号切换功能
提供数据备份和恢复功能
4.2 数据共享
我们不会与任何第三方共享您的数据
我们不会将数据发送到任何服务器
所有数据处理都在您的本地设备上完成
5. 数据安全
5.1 加密保护
所有敏感数据使用AES-GCM加密算法加密
加密密钥由用户主密钥派生
主密钥仅存储在您的本地设备上
5.2 访问控制
扩展需要用户输入主密钥才能解锁
未解锁状态下，无法访问加密数据
6. 权限说明
本扩展需要以下权限，用途如下：

storage：用于存储用户配置和会话数据
cookies：用于管理多账号的Cookie隔离
activeTab：用于在活动标签页中注入脚本，实现存储隔离
alarms：用于定时清理过期的Cookie
tabs：用于管理标签页和实例绑定关系
host_permissions：用于访问目标网站，实现Cookie注入和存储代理
7. 第三方服务
本扩展不使用任何第三方分析服务、广告服务或其他第三方服务。

8. 数据导出和删除
8.1 数据导出
您可以通过扩展的设置页面导出所有数据：

导出为JSON格式文件
包含所有模板、实例和配置数据
8.2 数据删除
您可以随时：

删除单个模板或实例
清除所有数据（通过扩展设置）
卸载扩展以删除所有数据
9. 儿童隐私
本扩展不面向13岁以下的儿童。我们不会故意收集儿童的个人信息。

10. 隐私政策更新
我们可能会不时更新本隐私政策。重大变更时，我们会在扩展中通知您。继续使用扩展即表示您接受更新后的隐私政策。
